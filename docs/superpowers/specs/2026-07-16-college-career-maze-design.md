# 大学生职业规划引导平台 — 产品设计文档

> 生成日期：2026-07-16
> 设计方法：结构化头脑风暴 + 深度调研（多源交叉验证）

---

## 一、问题定义

### 1.1 核心痛点

中国大学生（尤其是大一大二学生）面临系统性的规划困境：

- **信息不对称**：不知道考研/就业/考公/出国每条路的真实代价、回报、生活状态
- **信息失真**：幸存者偏差泛滥（成功者才发帖）、贩卖焦虑内容横行、AI 算法放大极端叙事
- **优绩主义内卷**：第一学历歧视、精英叙事的双重夹击——「只要你努力什么都能做到」vs「你没考上好大学你这辈子就完了」
- **无法倒推行动**：即使模糊有目标，也拆不成可执行的阶梯计划
- **路径单一**：大多数学生只知道考研和就业两条路，对低成本留学、自由职业、选调生等路径完全不了解

### 1.2 因果链

```
信息不对称（B）
  → 无法判断每条路（不知道考研和就业的真实代价差异）
    → 无法倒推行动计划（大一该干嘛？大二暑假该干嘛？）
      → 不知道自己想要什么（A：兴趣/价值观/长期目标不清晰）
        → 执行无力（D：拖延、知行不合一，只是表象）
```

### 1.3 市场空白（三重真空，经调研确认）

| 真空类型 | 具体情况 |
|---------|---------|
| **功能真空** | 无一产品同时提供「评估 + 数据 + 引导 + 社区」四层能力 |
| **信息质量真空** | 所有众包平台（Glassdoor/牛客/小红书）都存在幸存者偏差，无一做出系统性的反偏差治理 |
| **文化真空** | 无一产品正面应对中国市场结构性问题（第一学历歧视、优绩主义、信息茧房） |

---

## 二、产品定义

面向中国大学生的 **引导式决策平台**，帮他们在信息混乱的环境中看清每条路的真面貌，找到自己的方向，做出理性的、多维度的规划。

---

## 三、核心理念

| 理念 | 含义 | 设计原点 |
|------|------|---------|
| **H-I-P（人主导规划）** | AI 只是信息检索和推理工具，所有决策由人做出 | 学生保留最终决策权，AI 不说「你应该选A」，而是说「A路适配度72%，主要风险X」 |
| **三层同心圆** | 意愿层 ∩ 能力层 ∩ 现实层 → 最大交集 | 尊重学生「想成为什么样的人」，同时呈现真实约束 |
| **双通道漏斗** | 硬约束一票否决 → 偏好权重排序，每步可回溯 | 硬约束不是终点，被筛掉的选项可展开查看「子路径」 |
| **时滞校正** | 推荐数据对应「毕业年」而非「当下」 | 标注趋势箭头（上升/稳定/下降/替代风险），结合产业报告+政策文件 |
| **反幸存者偏差** | 每条数据标注来源+可信度+是否为极端个例 | 三色可信度系统（🟢官方公开数据/🟡AI推理数据/🔴社区未审核） |
| **路径丰富度保证** | 任何输出必须展示 3 条以上不同类别的路径 | 主动暴露学生不知道的选项，标注每条路径的「排他性」 |

## 四、架构设计

### 4.1 三层洋葱结构

```
┌──────────────────────────────────────────────┐
│          规划工具层（Human 操作）              │
│                                              │
│  路径对比器 · 决策画布 · 阶段性目标拆解工具     │
│  路径发现中心 · 规划文档生成                  │
│                 ↕ 按需调用                    │
├──────────────────────────────────────────────┤
│          AI 推理引擎层                        │
│                                              │
│  仅从知识库检索，不做开放域搜索                │
│  反事实解释 · 路径排序 · 风险标注 · 偏好提取  │
│                 ↕ 检索                        │
├──────────────────────────────────────────────┤
│          知识库层（可追溯可验证）              │
│                                              │
│  公共数据（O*NET CC-BY + 教育部报告 + 统计局）  │
│  AI 翻译/结构化/时滞校正                      │
│  种子内容（30 条核心路径）                    │
│  社区贡献 → 审核 → 入库（Wiki 模式演化目标）   │
│  每条数据：来源链接 + 可信度 + 更新时间        │
└──────────────────────────────────────────────┘
```

### 4.2 知识库数据策略

| 阶段 | 策略 | 规模 |
|------|------|------|
| **MVP** | O\*NET CC-BY 数据 + 中国教育部公开数据 + AI 翻译/结构化，人工审核 | 30 条核心路径，每条可信可追溯 |
| **中期** | 社区贡献入口开放 + 投票机制 + 统计异常检测（O\*NET 9% 淘汰率思路）| 覆盖主要专业 × 主要路径 |
| **成熟** | Wiki 全开放 + 爬虫自动化 + 多层审核 + 声誉系统 | 全面覆盖 |

## 五、决策引擎设计

### 5.1 九维硬约束（一票否决层，社区可扩展）

1. **经济可行性** — 总投入 vs 家庭预算，奖学金覆盖率
2. **语言能力** — 该路径的最低语言要求，达标所需时间
3. **学历准入** — 硬门槛/软门槛，你的第一学历的真实竞争力
4. **毕业难度** — 真实毕业率（社区反馈），延毕风险
5. **地域锁定** — 目标城市（如上海）的岗位充足度
6. **时间成本** — 到稳定就业的总时间线，年龄敏感度
7. **生活水平底线** — 在读/就业初期的可接受最低生活标准
8. **学历含金量** — 在目标城市/行业的真实认可度
9. **地缘政治与人身安全** — 国家关系、签证政策趋势、争议地区标记

> 所有约束不是死亡判决——被筛掉后系统标注「被哪个约束毙掉的」，学生可回看并展开子路径。

### 5.2 偏好权重层（排序不排除）

- 兴趣匹配度（计算机相关程度）
- 时间弹性（996 vs 弹性 vs 自由职业）
- 生活方式兼容（如：是否有时间参加漫展）
- 长期成长曲线（薪资增长、晋升空间）

### 5.3 输出格式

```
路径 A：★★★★ 外企软件开发
  ├─ 适配度：72%  |  硬约束通过数：8/9
  ├─ 优势：时间弹性好、上海岗位充足、学历要求相对宽松
  ├─ 风险：晋升天花板可能受学历限制、AI 辅助开发可能压低初级需求
  ├─ 趋势：稳定 ↑
  ├─ 排他性：基本放弃大厂应届高薪
  └─ 大一→大四执行阶梯：[展开]

路径 B：★★★☆ 国内读研 → 上海外企
  ├─ 适配度：68%  |  硬约束通过数：8/9
  ├─ ...
```

### 5.4 考研/读研特殊处理

- **不默认推荐读研**：读研是需要被论证的结论，不是起点
- **考研失败 Plan B**：永远并行展示
- **路径矩阵对比**：考研 vs 不考研的完整代价对比

## 六、方法论支撑（经学术调研确认）

| 方法 | 角色 | 来源 |
|------|------|------|
| **CIP CASVE 循环** | 用户交互主线（引导→分析→综合→评估→执行）| 认知信息加工理论 |
| **EBA 漏斗筛选** | 硬约束层的底层筛选逻辑 | Tversky (1972)；PIC 模型(Gati) |
| **联合分析** | 替代问卷，从对比选择中反推真实偏好 | MIT 1000+ 学生随机实验验证 |
| **反事实可解释推荐** | 结果展示：为什么这条路在这里，优势和风险是什么 | 85 篇综述结论：E-P 评分最高(0.95) |
| **SCCT** | 建设职业决策自我效能感（比给答案更重要）| SVEI 模型已验证 |

## 七、MVP 范围

### 7.1 包含

- Web 应用（响应式）
- 知识库：30 条核心路径 × 9 维硬约束 × 趋势标签
- AI 对话引导（接入 LLM，仅从知识库 RAG 推理）
- 路径对比器（最多同时对比 5 条路径）
- 可信度三色标注
- 大一→大四阶梯计划生成
- GitHub 开源 + 社群分发

### 7.2 不含（后续迭代）

- 小程序（中期）
- 社区贡献系统
- Wiki 模式知识库
- 自动爬虫更新

## 九、演化路径

```
MVP（个人开发者）
  ├─ 公共开放数据 + AI 翻译/结构化 + 人工审核
  ├─ 30 条核心路径
  ├─ 发布到社群 + GitHub 开源
  └─ 目标：验证核心体验是否真正帮到人

中期（有用户基础）
  ├─ 社区贡献入口 + 投票 + 统计异常检测
  ├─ 覆盖主要专业 × 主要路径
  └─ 目标：知识库从「我维护」变成「大家维护，我审核」

成熟（有团队/收入）
  ├─ Wiki 全开放 + 爬虫自动化 + 多层审核 + 声誉系统
  └─ 目标：成为中国大学生规划的事实标准
```

## 十、约束与风险

| 风险 | 缓解策略 |
|------|---------|
| 个人开发者资源有限 | 借力 O\*NET CC-BY + 公开数据 + AI 工具链，不求大而全，求少而精 |
| AI 幻觉/可信度 | 知识库 RAG 限制 + 来源强制标注 + 每句断言可追溯到源数据 |
| 冷启动无用户 | GitHub 开源 + 社群分发，靠质量和诚意吸引种子用户 |
| 内容法律风险 | 不提供「权威建议」，始终保持「信息呈现+引导思考」的定位 |
| 数据时效性 | 事件驱动更新 + 每条数据标注最后更新时间 + 趋势箭头标注 |
