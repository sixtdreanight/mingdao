# 明道 AI 规划师系统重设计

> 版本: v2.0 | 日期: 2026-07-17 | 状态: 待确认

## 产品闭环

```
个人画像 ←→ AI 规划师 ←→ 路线规划
    ↓                      ↓
  信息采集               路线推荐
    ↓                      ↓
    └──── 数据库 + 实时搜索 ────┘
                ↓
           学习资源匹配
                ↓
           成就追踪系统
```

## 1. AI 规划师：三阶段机

### 阶段一：信息采集

AI 像心理咨询师一样逐步了解用户。

**行为**：
- 一次只问一个维度的缺失信息
- 先确认上一轮回答，再问下一个
- 解释为什么需要这个信息
- 学生说「不知道」→ 记为空，不追问

**触发规划的条件**：画像 >= 6 维 或 学生主动说「帮我规划一下」

### 阶段二：路线规划

画像足够时，查询数据库 + 实时搜索，生成 2-4 条路线。

每条路线：
```typescript
interface Route {
  id: string;
  title: string;         // "人工智能算法岗（大厂路线）"
  overview: string;      // 一句话概述
  fit: string;           // 适合什么样的人
  requirements: string[]; // 门槛条件
  timeline: RouteStep[];  // 分阶段执行
  cost: string;          // 时间/金钱代价
  salary: string;        // 薪资范围（带来源链接）
  tags: string[];        // ["高薪", "竞争激烈", "需硕士"]
  resources: ResourceMatch[]; // 匹配的学习资源
}
```

### 阶段三：调整确认

学生反馈 → AI 排除不合适路线，补充新路线，或深入某一条。
学生说「就这个」→ 路线锁定为「当前目标」，进入成就追踪。

## 2. 数据库 + 实时搜索

```
用户提问 → 实时搜索（DuckDuckGo） → 真实数据 + 来源链接
          → 本地知识库               → 结构化薪资/教育/政策数据
          → 用戶画像                 → 个性化过滤
```

- 搜索结果是唯一真实数据来源
- 本地知识库存放经过验证的结构化数据
- 所有数据引用必须标注来源

## 3. 学习资源匹配

每条路线的每一步（如「考研」「刷题」「实习」）匹配对应的学习资源：

```
路线步骤「考985人工智能硕士」→ 匹配资源：
  - 中国大学MOOC：CS考研专业课
  - 王道考研：408计算机学科基础
  - 报考院校官网
```

资源来自已有的 `RESOURCE_INDEX`（310+ 链接）+ AI 搜索补充。

## 4. 个人画像（可编辑）

学生可随时手动修改画像信息：
- 年级、专业、学校
- 目标城市
- 预算
- 兴趣偏好
- 生活方式
- 底线红线
- 当前能力（自评 1-5）

画像更新后 → AI 重新评估阶段 → 在规划阶段的路线随画像调整。

## 5. 成就链图（科技树式可视化）

每条路线展开为一个用线连接起来的节点链，像游戏科技树或地铁线路图。

### 多条路线并排

```
  路线A：大厂算法              路线B：AI产品经理

  ⭐ 校招SP                   ⭐ 大厂PM offer
   │                           │
  ⏳ 实习                      ⏳ 产品实习
   │                           │
  ✅ 刷题                      ✅ 项目经验
   │                           │
  ✅ 考研985                   🔒 考证
   │                           │
  📍 起点                      📍 起点
```

每条路线是一条独立的链。学生可以同时追踪多条。

### 交互

- **点击节点** → 展开详情（做什么、怎么判断完成、匹配的资源）
- **勾选完成** → 节点变绿，后续节点解锁
- **新路线生成时** → 新链从起点长出，动画展开
- **画像变更** → 受影响的路线上出现 ⚠️ 标记，「这条路线的条件变了，点击查看」

### 数据模型

```typescript
interface RouteTree {
  id: string;
  title: string;
  status: 'active' | 'completed' | 'abandoned';
  nodes: TreeNode[];
  edges: TreeEdge[];     // { from: nodeId, to: nodeId }
}

interface TreeNode {
  id: string;
  label: string;         // "考研上岸985 CS"
  status: 'locked' | 'active' | 'done' | 'goal';
  x: number; y: number;  // 画布坐标
  resources?: ResourceLink[];
}

interface TreeEdge {
  from: string;
  to: string;
}
```

### 技术实现

用 SVG 渲染——节点是圆角矩形 + 图标，边是贝塞尔曲线连线。
布局算法：拓扑排序，根节点在底部，目标在顶部。
已存在的库：React Flow（节点图渲染）可作为参考，但我们的图是树状结构更简单。

## 7. 开发顺序

1. 阶段机 + 路线生成器（后端核心）
2. 路线卡片 + 成就看板（前端展示）
3. 画像可编辑
4. 资源匹配
5. 成就追踪（勾选/随画像调整）
