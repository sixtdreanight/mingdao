# 明道 AI 规划师系统重设计

> 版本: v2.0 | 日期: 2026-07-17 | 状态: 待确认

## 产品闭环

```
个人画像 ←→ AI 规划师 ←→ 路线规划
    ↓                      ↓
  信息采集               路线推荐
    ↓                      ↓
    └──── 数据库 + 实时搜索 ────┘
                ↓
           学习资源匹配
                ↓
           成就追踪系统
```

## 1. AI 规划师：三阶段机

### 阶段一：信息采集

AI 像心理咨询师一样逐步了解用户。

**行为**：
- 一次只问一个维度的缺失信息
- 先确认上一轮回答，再问下一个
- 解释为什么需要这个信息
- 学生说「不知道」→ 记为空，不追问

**触发规划的条件**：画像 >= 6 维 或 学生主动说「帮我规划一下」

### 阶段二：路线规划

画像足够时，查询数据库 + 实时搜索，生成 2-4 条路线。

每条路线：
```typescript
interface Route {
  id: string;
  title: string;         // "人工智能算法岗（大厂路线）"
  overview: string;      // 一句话概述
  fit: string;           // 适合什么样的人
  requirements: string[]; // 门槛条件
  timeline: RouteStep[];  // 分阶段执行
  cost: string;          // 时间/金钱代价
  salary: string;        // 薪资范围（带来源链接）
  tags: string[];        // ["高薪", "竞争激烈", "需硕士"]
  resources: ResourceMatch[]; // 匹配的学习资源
}
```

### 阶段三：调整确认

学生反馈 → AI 排除不合适路线，补充新路线，或深入某一条。
学生说「就这个」→ 路线锁定为「当前目标」，进入成就追踪。

## 2. 数据库 + 实时搜索

```
用户提问 → 实时搜索（DuckDuckGo） → 真实数据 + 来源链接
          → 本地知识库               → 结构化薪资/教育/政策数据
          → 用戶画像                 → 个性化过滤
```

- 搜索结果是唯一真实数据来源
- 本地知识库存放经过验证的结构化数据
- 所有数据引用必须标注来源

## 3. 学习资源匹配

每条路线的每一步（如「考研」「刷题」「实习」）匹配对应的学习资源：

```
路线步骤「考985人工智能硕士」→ 匹配资源：
  - 中国大学MOOC：CS考研专业课
  - 王道考研：408计算机学科基础
  - 报考院校官网
```

资源来自已有的 `RESOURCE_INDEX`（310+ 链接）+ AI 搜索补充。

## 4. 个人画像（可编辑）

学生可随时手动修改画像信息：
- 年级、专业、学校
- 目标城市
- 预算
- 兴趣偏好
- 生活方式
- 底线红线
- 当前能力（自评 1-5）

画像更新后 → AI 重新评估阶段 → 在规划阶段的路线随画像调整。

## 5. 成就追踪

每条 AI 生成的路线自动保存为「成就卡」：

```
┌─ 成就卡 ─────────────────────────┐
│ 🎯 人工智能算法岗（大厂路线）      │
│ ██████████░░ 30% 达成             │
│                                  │
│ ☑ 考上985/211硕士                 │
│ ☐ 算法刷题200道                   │
│ ☐ 2段实习经历                     │
│ ☐ 校招SP offer                   │
│                                  │
│ 生成时间：2026-07-17              │
│ 最后更新：2026-07-17              │
└──────────────────────────────────┘
```

- 每步可手动勾选完成
- 画像更新时成就卡自动调整（如换了专业→路线可能需要重来）
- 多张成就卡可并存（多条路线同时追踪）
- 存储在 localStorage

## 6. 技术架构

```
src/
├── lib/
│   ├── session.ts          # 阶段机：判断阶段 + 引导提问
│   ├── planner.ts           # 路线生成器
│   ├── route-store.ts       # 路线/成就 localStorage 管理
│   ├── web-search.ts        # 实时搜索（已有）
│   ├── ai-client.ts         # AI 客户端（已有）
│   ├── ai.ts                # 聊天逻辑（重写）
│   └── rag.ts               # 数据层（简化）
├── components/
│   ├── chat/ChatInterface.tsx    # 聊天（已有）
│   ├── profile/ProfileDashboard.tsx  # 画像编辑
│   ├── profile/ProfileEditor.tsx     # 手动编辑表单
│   ├── routes/RouteCard.tsx          # 单条路线卡片
│   ├── routes/AchievementBoard.tsx   # 成就看板
│   └── routes/RouteStepList.tsx      # 分步任务列表
├── app/api/
│   ├── chat/route.ts          # 聊天 API（已有）
│   └── plan/route.ts          # 路线生成 API（新增）
```

## 7. 开发顺序

1. 阶段机 + 路线生成器（后端核心）
2. 路线卡片 + 成就看板（前端展示）
3. 画像可编辑
4. 资源匹配
5. 成就追踪（勾选/随画像调整）
