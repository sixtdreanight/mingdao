# 安全策略

## 报告漏洞

如果您发现安全漏洞，**请不要**创建公开的 Issue。

请通过 GitHub Security Advisories 私下报告，或发送邮件至项目维护者。

我们将在 48 小时内确认收到您的报告，并在 7 天内提供修复时间表。

## 支持版本

| 版本 | 支持 |
|------|------|
| 0.1.x | ✅ |

## 安全最佳实践

本项目遵循以下安全最佳实践：

- 所有 API 密钥通过环境变量管理，永不硬编码
- 用户输入在服务端验证
- API 响应不泄露内部错误细节
- 外部链接使用 `rel="noopener noreferrer"`
- 依赖项定期审查和更新
